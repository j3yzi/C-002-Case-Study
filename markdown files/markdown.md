  <summary><span style="font-size: 20px; font-weight:700; color: #000;"> 🛠️ Infrastructure Layer Components</span></summary>

    <tr>
      <td>
        <h4>🔧</h4>
      </td>
      <td>

        <ul>
          <li>Configuration management for business rules</li>

          <li>Form validation with multiple validation types</li>
          <li>Terminal operations for console manipulation</li>
        </ul>

    float regularHours;    // Configurable payroll settings
    float overtimeRate;
    float passingGrade;    // Configurable academic settings

      <td align="center">
        <h4>🗂️</h4>
      </td>

          <li>Type-agnostic lists supporting any data type</li>
          <li>Multiple list types (singly, doubly, circular)</li>
          <li>Memory management with custom free functions</li>
          <li>Core operations: add, remove, search, traverse</li>
        </ul>
      </td>
    </tr>
    <tr>
      <td align="center">
        <h4>🌐</h4>
      </td>

extern appState studentListCreated;</code></pre>

    </tr>

      <td align="center">

      </td>

<details>
  <summary><span style="font-size: 20px; font-weight:700; color: #000;">🧩 Core Programming Principles in C</span></summary>
  
  <h4>Modular Programming</h4>
  <p>We strictly followed the best practice of separating interfaces from implementations:</p>
  
  <table style="width: 700px; border-collapse: collapse; margin-top: 30px; justify-content: center;">
    <tr>
      <td width="50%">
        <strong>Header Files (.h)</strong>
        <p>Define the "what" - function prototypes, struct definitions, and enums</p>
        <p><code>include/headers/</code>, <code>include/models/</code>, <code>src/ui/</code>, <code>src/modules/</code></p>
      </td>
      <td width="50%">
        <strong>Source Files (.c)</strong>
        <p>Define the "how" - actual code and logic for the functions</p>
        <p><code>include/src/</code>, <code>src/ui/</code>, <code>src/modules/</code></p>
      </td>
    </tr>
  </table>
  
  <h4>Safety and Robustness</h4>
  <table style="width: 700px; border-collapse: collapse; margin-top: 30px; justify-content: center;">
  <thead>
    <tr style="background-color: #f0f0f0;">
      <th style="padding: 12px; text-align: left;">📌 Code Quality Practice</th>
      <th style="padding: 12px; text-align: left;">📝 Description</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td style="padding: 10px;"><strong>Safe Input Handling</strong></td>
      <td style="padding: 10px;">Replaced <code>scanf()</code> with <code>fgets()</code> + <code>sscanf()</code> to prevent buffer overflows</td>
    </tr>
    <tr style="background-color: #f9f9f9;">
      <td style="padding: 10px;"><strong>Memory Management</strong></td>
      <td style="padding: 10px;">Strict use of <code>malloc()</code> and <code>free()</code>, including custom free functions for cleanup</td>
    </tr>
    <tr>
      <td style="padding: 10px;"><strong>Const Correctness</strong></td>
      <td style="padding: 10px;">Used <code>const</code> on function parameters that should not be modified</td>
    </tr>
    <tr style="background-color: #f9f9f9;">
      <td style="padding: 10px;"><strong>Minimal Global Variables</strong></td>
      <td style="padding: 10px;">Avoided global variables to reduce side effects and improve modularity</td>
    </tr>
  </tbody>

  <h4>Sorting Algorithm Research</h4>
  <p>For sorting operations, we compared multiple algorithms:</p>
  
  <table style="width: 700px; border-collapse: collapse; margin-top: 30px; justify-content: center;">
    <tr>
      <th>Algorithm</th>
      <th>Time Complexity</th>
      <th>Space Complexity</th>
      <th>Suitability for Linked Lists</th>
    </tr>
    <tr>
      <td>Bubble Sort</td>
      <td>O(n²)</td>
      <td>O(1)</td>
      <td>Simple but inefficient for large lists</td>
    </tr>
    <tr>
      <td>Merge Sort</td>
      <td>O(n log n)</td>
      <td>O(n)</td>
      <td>Excellent - doesn't require random access</td>
    </tr>
    <tr>
      <td>Quick Sort</td>
      <td>O(n log n) average<br>O(n²) worst case</td>
      <td>O(log n)</td>
      <td>Less suitable - requires random access</td>
    </tr>
  </table>
  
  <p>For the initial implementation, we used <strong>Bubble Sort</strong> for simplicity, with plans to upgrade to <strong>Merge Sort</strong> for better performance with larger datasets.</p>
